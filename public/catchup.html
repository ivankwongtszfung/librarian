Key decisions & why
ADR-003 — record vs mailbox: the decision record never leaves local SQLite; cross-device sync is a transient ciphertext-only mailbox. Multi-device forces a store — devices aren't co-present; “pure tunnel” was retired for that reason.

ADR-004 — a verdict authenticates itself: an approved makes an agent act, so verdict authority must never be grantable by anything a transport holds (token, JWT, mailbox possession). The gate for all remote exposure.

ADR-005 — clean architecture, lightly: the Dependency Rule (domain ← application ← infrastructure/interfaces), no DTO-per-layer ceremony. The DecisionStore port is the seam new storage/transports plug into.

ADR-007 — accept the dev channel locally: --dangerously-load-development-channels is the supported path for a self-registered channel, not a warning about the code; the trade is bounded by the loopback gate above.

ADR-008 — provenance is additive, authority is not: only a verdict event may make a decision approved. Dedup half shipped (PR #10); watcher half approved & awaiting build.

ADR-009 — mermaid over ASCII: diagrams are data, not art; vendored (3.4 MB) + pinned because the review UI is the verdict surface and loads zero remote code.

Invariant: a red light without a reason is not a decision — enforced in the state machine, the schema, and the API.
